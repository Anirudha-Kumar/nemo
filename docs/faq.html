An F.A.Q.

Using NEMO

  1. How do i get NEMO started/loaded
    Either add to your .cshrc C-shell startup file, or interactively use the command 
    	source /NEMO/nemo_start
    or where-ever the root directory of NEMO may be. But it should contain the C-shell compatible NEMO startup commands.
  2. Help, how do i get help
    Most NEMO programs have (standard unix) manual pages, but each program can also describe itself using the help= system keyword. And there is of course the web 
        % man programs
    % man index
    % man snapplot				(or whatever program)
    % ghostview $NEMO/text/manuals/nemo.ps
    % tsf help=					(or whatever program)
    % tsf help=\?				(or whatever program)

Installing NEMO

  1. I have an installation of nemo which, for some reason, I want to re-compile with a version of gcc which is not in the standard path. How can I do this? Can I avoid re-installing nemo from scratch?
    You could just edit the macros CC, CXX and FC in the $NEMOLIB/makedefs file, but when you ran configure, it probably also constructed lots of other macros in that file. So, basically, the procedure would be: 
        cd $NEMO
    configure  
    make postconfig
    diff NEMORC.local NEMORC.gen            <-- study the differences and see if relevant
    source NEMORC.local                     do this if you decided to edit it via NEMORC.gen
    make libs                               recompile library, study install.log if it's ok
    src/scripts/testsuite -b                recompile a bunch of basic binaries
    mknemo                        recompile other programs you may need

Programming NEMO

  1. Compiling and adding a "standard" program 
        % cd $NEMO/src/.....        (choose some reasonable place)
    % edit program.c
    % bake program          	(local build)
or
    % mknemo program        	(compiles and copies to $NEMOBIN)
  2. Adding something new (but "standard") to the library 
        % cd $NEMO/src/.....        (choose some reasonable place)
    % edit Makefile             (add the .o file to the object list)
    % make install              (compile and put in $NEMOLIB/libnemo.a)
